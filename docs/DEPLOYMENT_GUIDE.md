# Frontend Deployment Guide

The frontend deploys as static assets generated by Create React App.

## Build Command

```bash
npm run build
```

Output directory:

```text
build/
```

## Production Environment

Set these in the hosting provider:

```env
REACT_APP_API_URL=https://digital-logics-studio-backend.vercel.app/api
REACT_APP_SITE_URL=https://circuits.quantumlogicslimited.com
REACT_APP_GA_MEASUREMENT_ID=
REACT_APP_GOOGLE_SITE_VERIFICATION=
REACT_APP_BING_SITE_VERIFICATION=
```

The backend must also allow the frontend origin through `CLIENT_URL`, `CLIENT_ORIGIN`, or `FRONTEND_URL`.

## Static Hosting Requirements

Because the app uses React Router, all unknown paths must rewrite to `index.html`.

Recommended Vercel behavior:

- Serve `build/`.
- Rewrite SPA routes to `index.html`.
- Keep generated `robots.txt`, `sitemap.xml`, and static prerendered pages available.

## SEO Build Artifacts

The build process generates or updates:

- `public/sitemap.xml`
- `public/robots.txt`
- `react-snap-routes.json`
- prerendered HTML snapshots under `build/`

Production deploys should include these artifacts from a fresh build, not stale local output.

## Release Checklist

1. Set production environment variables.
2. Run `npm test` for changed areas.
3. Run `npm run build`.
4. Verify `build/index.html` contains expected metadata.
5. Verify `build/sitemap.xml` and `build/robots.txt`.
6. Deploy frontend.
7. Confirm backend allows the deployed origin.
8. Test signup/login/logout.
9. Test profile route protection.
10. Test one progress action and page refresh.
11. Run Lighthouse on home, a topic page, and `/problems`.

## Rollback

Use the hosting provider's previous deployment rollback. If auth fails after rollback, verify frontend and backend environment variables are rolled back together.

## Production Monitoring

Recommended signals:

- Client-side route errors.
- Failed API responses by route.
- Signup/login conversion.
- Progress sync failures.
- Web vitals.
- Search Console crawl/indexing issues.
